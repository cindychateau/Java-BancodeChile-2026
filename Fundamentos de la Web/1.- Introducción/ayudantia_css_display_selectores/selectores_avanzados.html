Selectores Avanzados CSS

Aprenderemos el selector comodín, descendente, descendente directo, combinados y de atributo.

1. Selector Comodín (*)

El asterisco * selecciona TODOS los elementos de la página. Es el selector más general que existe y tiene la especificidad más baja.

Es útil para aplicar un estilo base a todo, como quitar márgenes o definir el box-sizing.

2. Especificidad: ¿qué selector gana?

Cuando varios selectores apuntan al mismo elemento, gana el más específico. La "fuerza" de cada selector:

  • Selector comodín * → fuerza 0
  • Selector de etiqueta (h1, p…) → fuerza 1
  • Selector de clase (.verde…) → fuerza 10
  • Selector de ID (#rojo…) → fuerza 100

¿De qué color soy?

El h1 de arriba tiene a la vez: etiqueta h1 (azul), class="clase-verde" (verde) e id="id-rojo" (rojo). El ID gana porque tiene más fuerza (100).

3. Selector Descendente (padre hijo)

Selecciona un elemento que esté en cualquier nivel dentro de otro. Se escribe con un espacio entre padre e hijo.

Ejemplo: .cita p selecciona todos los p dentro de .cita, sin importar cuántos niveles haya entre ellos.

4. Selector Descendente Directo (padre > hijo)

El símbolo > selecciona solo los hijos directos, no los nietos ni más profundo.

Ejemplo: .menu > li solo afecta a los li que están directamente dentro de .menu.

5. Selectores Combinados

Podemos combinar tipo + clase, o ID + etiqueta, para ser más específicos y no afectar a otros elementos.

Ejemplo: #contenedor p.destacado solo afecta a los p con clase "destacado" que estén dentro de #contenedor.

6. Selector de Atributo ([atributo])

Selecciona elementos según el valor de sus atributos HTML. Muy útil para estilizar formularios.

Ejemplo: input[type="text"] solo afecta a los input de tipo texto.
